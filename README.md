# FinanceTracker

Мульти-бот система на Python для учета финансов с поддержкой:
- Telegram
- MAX
- SQLite
- Docker Compose

## Что умеет сейчас
- Главное меню и пошаговый пользовательский интерфейс
- Всегда доступны переходы `Назад` и `Главное меню`
- Добавление операции по шагам с проверкой типов
- Кнопочный выбор даты операции и категорий
- Для сумм из сообщения извлекается только число
- Добавление категории по шагам
- Редактирование и удаление операций по шагам
- Баланс
- Отчеты за день, месяц, год
- Лимиты бюджета на месяц
- Напоминания в заданное время
- Журнал последних операций
- Общая бизнес-логика для Telegram и MAX

## Интерфейс
### Telegram
- кнопки для меню и шагов
- кнопочный выбор типа операции, периода, даты и категорий
- пагинация категорий

### MAX
- нативные кнопки через клавиатуру MAX Bot API
- те же пошаговые сценарии, что и в Telegram
- кнопочный выбор типа операции, периода, даты и категорий
- пагинация категорий

## Быстрый старт
1. Скопируйте `.env.example` в `.env`
2. Заполните токены Telegram и MAX
3. Убедитесь, что у MAX бота отключен webhook, так как проект использует polling
4. Запустите:

```bash
docker compose up -d --build
```

## Хранение данных
- SQLite база: `./data/finance_tracker.db`
- Экспорт/служебные файлы: `./exports`

## Структура проекта
```text
app/

  core/

```

## Особенности валидации
- для ввода сумм бот ищет в сообщении первое число и использует только его
- отрицательные и нулевые суммы отклоняются
- время напоминания проверяется строго в формате `ЧЧ:ММ`
- на каждом шаге можно вернуться назад или в главное меню
